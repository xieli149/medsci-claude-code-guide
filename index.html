Part 01 · Install

先完成 Claude Code 安装

01

打开 PowerShell

点击开始菜单，搜索“PowerShell”并打开普通窗口。无需“以管理员身份运行”。

看一眼窗口开头：出现 PS C:\Users\你的名字>，说明打开正确。
02

检查 Node.js 版本

先运行下面两条命令。如果第一条显示 v22 或更高版本，直接进入下一步。

PowerShell
node -v
npm -v
没有版本号或低于 v22？不要进入文件列表，直接按电脑类型点击下面的安装包。
绝大多数 Windows 电脑选这个Windows 普通电脑（x64）→适合 Intel 或 AMD 处理器的 Windows 电脑。node-v22.23.1-x64.msi Windows ARM 电脑 →仅适合骁龙 Snapdragon 等 ARM 处理器电脑。node-v22.23.1-arm64.msi
不确定就选第一个 x64。只有“设置 → 系统 → 系统信息 → 系统类型”明确写着“基于 ARM 的处理器”时，才选 ARM64。不要下载 x86、zip、7z 或 tar.gz。
双击下载的 .msi，安装时一路保留默认选项。完成后关闭 PowerShell，再重新打开，然后再次检查版本。
03

切换国内镜像并安装

不要一次复制两条。先复制并执行第 1 条，完成后再复制并执行第 2 条。

第 1 条 · 切换国内镜像
npm config set registry https://registry.npmmirror.com
第 2 条 · 安装 Claude Code
npm install -g @anthropic-ai/claude-code@latest
只安装这个官方包名：@anthropic-ai/claude-code。不要安装名字相似的第三方包。
04

验证安装结果

能看到版本号，就代表 Claude Code 程序本身已经安装成功。

PowerShell
claude --version
claude doctor
说明：doctor 可能提示尚未登录，这是正常的。下一部分会用硅基流动 API 代替 Claude 官方账号。
01

打开“终端”

按 Command + 空格 打开聚焦搜索，输入“终端”或“Terminal”，回车打开。

02

检查 Node.js 版本

如果第一条显示 v22 或更高版本，直接进入下一步。

Terminal
node -v
npm -v
没有版本号或低于 v22？macOS 只有一个通用安装包，Apple 芯片和 Intel 芯片都可以直接使用。
所有 Mac 都选这个下载 macOS 通用安装包 →兼容 Apple M1/M2/M3/M4/M5 芯片与 Intel 芯片。node-v22.23.1.pkg
只下载上面的 .pkg。不要下载 darwin、tar.gz、tar.xz 等压缩包。双击 PKG 后一路按默认选项安装。
03

设置个人安装目录与国内镜像

先把全局工具放到自己的用户目录，避免 macOS 权限报错；再切换国内下载源。

Terminal · 可整段复制
mkdir -p "$HOME/.npm-global"
npm config set prefix "$HOME/.npm-global"
grep -q '.npm-global/bin' "$HOME/.zshrc" 2>/dev/null || echo 'export PATH="$HOME/.npm-global/bin:$PATH"' >> "$HOME/.zshrc"
export PATH="$HOME/.npm-global/bin:$PATH"
npm config set registry https://registry.npmmirror.com
04

安装并验证 Claude Code

不要在命令前添加 sudo。

Terminal
npm install -g @anthropic-ai/claude-code@latest
claude --version
claude doctor
说明：doctor 可能提示尚未登录，这是正常的。下一部分会接入硅基流动。
Part 02 · Connect API

接入硅基流动 API

培训使用硅基流动，不需要 Claude 官方订阅。通过 CC Switch 的 SiliconFlow 预设接入，少改配置，也方便以后切换模型。

05

注册账号并领取可用额度

使用培训专属链接进入硅基流动，完成注册与平台要求的认证。注册链接附带活动额度，实际到账以注册页与账户余额显示为准。

硅基流动培训注册链接注册完成后，在控制台右上角进入“账户余额”确认额度。
注册 / 登录硅基流动 ↗
06

创建 API Key

进入“API 密钥”页面，点击“新建 API 密钥”。复制以 sk- 开头的密钥，先保存在本机安全位置。

API Key 是你的调用密码页面不会要求你把 Key 粘贴到这里；稍后只粘贴到 CC Switch。
打开 API 密钥页 ↗
不要泄露 API Key：不要发到群里、不要放进截图、不要写入代码或提交到 GitHub。泄露后应立即在控制台删除并新建。
07

安装 CC Switch

CC Switch 是开源的跨平台配置工具，硅基流动官方文档推荐用它为 Claude Code 添加 SiliconFlow 预设。

按电脑系统下载Windows 选 Windows.msi；macOS 选 macOS.dmg。如果 GitHub 较慢，可请培训讲师提供现场安装包。
下载 CC Switch ↗查看项目官网 ↗
macOS 首次打开被拦截：进入“系统设置 → 隐私与安全性”，找到 CC Switch，点击“仍要打开”。只从上面的项目官方链接下载安装。
08

添加并启用 SiliconFlow 预设

按顺序操作，整个过程不需要手动编辑配置文件。

  1. 1在 CC Switch 顶部应用切换器中选择 Claude。
  2. 2点击右上角 ＋，在“预设”中选择 SiliconFlow（紫色图标）。
  3. 3确认端点自动显示 https://api.siliconflow.cn，模型由预设自动填充。
  4. 4把刚才创建的 API Key 粘贴到 API Key，点击“添加”。
  5. 5回到供应商列表，在 SiliconFlow 卡片上点击 启用，看到蓝色边框和“当前启用”。
首次启动仍要求登录？在 CC Switch 打开“设置 → 通用”，开启“跳过 Claude Code 初次安装确认”，然后重新启动 Claude Code。
Part 03 · First Run

完成第一次真实任务

Claude Code 的工作对象是“当前文件夹”。先创建一个练习文件夹，再从这个文件夹启动。

PowerShell
mkdir "$HOME\Desktop\Claude练习"
cd "$HOME\Desktop\Claude练习"
claude
Terminal
mkdir -p "$HOME/Desktop/Claude练习"
cd "$HOME/Desktop/Claude练习"
claude
首次授权提示：Claude Code 会询问是否信任当前文件夹。这里只是自己新建的练习文件夹，可以确认信任；不要随意信任来源不明的代码目录。
启动后，把这段话发给 Claude Code
请在当前文件夹创建一个“梅斯医学 AI 工具体验”网页：包含蓝色标题、三个功能卡片和当前日期。完成后告诉我创建了哪些文件，以及如何打开查看。先不要安装任何额外软件。
Part 04 · Troubleshooting

遇到报错，先对照这里

安装问题和 API 问题要分开判断：能输出版本号代表程序已装好；能正常回答才代表 API 已接通。

提示 “claude 不是内部或外部命令” / command not found

先关闭当前终端并重新打开，再运行 claude --version。macOS 还可以运行 source ~/.zshrc 后重试。

npm 出现 EACCES / permission denied

macOS 不要使用 sudo。回到上面的“设置个人安装目录与国内镜像”，完整执行一次，再重新安装。

提示找不到 native binary package

通常是 npm 禁用了 optional dependencies。检查 .npmrc 中不要有 optional=false，然后重新运行安装命令，不要添加 --omit=optional。

启动后一直要求 Claude 登录

先确认 CC Switch 中 Claude → SiliconFlow 卡片已“当前启用”。再进入 CC Switch“设置 → 通用”，开启“跳过 Claude Code 初次安装确认”，关闭终端后重开。

出现 401 / invalid API key / 余额不足

401 通常表示 Key 填错或已失效：去硅基流动控制台重新复制。余额不足则先检查账户余额和模型计费；API Key 不要带多余空格。

Windows 上命令执行能力不完整

Claude Code 原生 Windows 可以使用 PowerShell。需要更完整的 Git/Bash 工作流时，可另行安装 Git for Windows；这不是本次完成安装与首次体验的必需项。

Before You Start

三条安全边界

🔑

API Key 只放本机

不发群、不截图、不提交 GitHub。怀疑泄露，立即删除并重建。

👁

执行前看清操作

Claude Code 能读写文件、运行命令。涉及删除、覆盖、安装时，先确认影响范围。

📁

从正确文件夹启动

它默认围绕当前文件夹工作。公司敏感资料不要放入未经批准的练习目录。
